### 1.1 솔버 검증 — Maxwell PDF & 실제 스크린샷 대조 (400W 모델)

400W CoBot 모터(18슬롯/16극 SPM, 설계 `4. 400W_BasicModel_..._Optimized`)를
기준으로 Python 솔버(motoropt)의 정확도를 항목별로 검증.

| 항목 | Maxwell | motoropt | 편차 |
|---|---|---|---|
| 설계변수 26종 (수식 포함) | Properties 창 | 전부 일치 | 0% |
| 재질 (Kh/Kc/Ke·점적 0.97·도전율·BH) | 재질 다이얼로그 | 전부 일치 | 0% |
| 무부하 Az 최대 | 0.0048 Wb/m | 0.0049 | +2% |
| 무부하 B 최대 | 2.309 T | 2.466 | +6.8% (점 최대값, 메시 민감) |
| EMF rms @1000rpm | 6.165~6.172 V | 6.161~6.164 | **−0.1%** |
| EMF 피크 | 8.37~8.51 V | 8.45 | 범위 내 |
| 코깅 pk2pk | 5.347 mNm | 6.87 | +28% |

- **EMF·형상·재질은 사실상 완전 일치.** 코깅만 과대평가가 약점.
- 코깅 +28%: Maxwell 스크린샷도 무부하 평균토크가 −1.778 mNm로 찍힘(참값 0이어야
  함) → Maxwell 쪽도 ~2 mNm 수치 노이즈 존재. 36스텝은 코깅 1주기당 2샘플로
  언더샘플링. 코깅은 절대값보다 "수 mNm 스케일 경향 지표"로 사용 권장.
- 손실 PDF 대조(이전): 토크 +2.9%, 철손 −21~25%(±25% 신뢰구간), 효율은 자석손
  가산 후 근접. 자석 재질에 도전율 555,556 S/m 확인 → Maxwell 자석손 2.2W는
  transient 와전류 계산값. 향후 자석 와류손 모델 추가 근거 확보.

검증 기준값·재질값은 메모리 `project-400w-pdf-reference.md`에 상세 기록.

### 1.2 GUI 개선 (이번 및 직전 세션)

- Objective 탭: 응답명에 **단위 표시** (`T_avg [mNm]`, `emf_rms [V]`,
  `magnet_area [mm²]`, `ripple_pct [%]`, `efficiency [0~1]`).
- 부하 해석 버튼이 **입력 전류를 사용**하도록 수정 (이전엔 모델 정격만 사용).
- 선간↔상전류 **실시간 환산 라벨** (Y/Δ 결선 구분, Δ → 선간/√3).
- venv 자동 재실행, 예외 다이얼로그(PyQt6 abort 방지), 모델별 DOE 데이터셋 분리.

## 2. 이번 세션에서 답한 질문들

### Q. larger/smaller인데 L·U가 둘 다 활성화되는 게 맞나?
**맞다.** Derringer-Suich는 합격/불합격이 아니라 0~1 만족도 점수라 두 점이 필요.
- larger: L 미만=0점(탈락), U 이상=만점(상한 제약 아님, "충분" 포화점)
- smaller: L 이하=만점, U 초과=0점
- ⚠️ KRO80에서 T_avg를 larger L=3000/U=4500으로 두면 기준 토크(8489)가 이미
  U를 넘어 **항상 만점** → 최적화가 토크를 안 지킴. 400W처럼 L=기준값,
  U=기준×1.03 로 두는 걸 권장.

### Q. KRO80 부하 토크 8700.9 mNm 맞나?
**맞다.** 교차검증:
- DOE 기준설계 Arkkio 9035 mNm(28.1A) → 26.9A 환산 ≈8650, 단일포지션 8701 +0.6%
- DOE 가상일 8489 mNm(28.1A) → 더 정확한 평균토크 ~8.1 Nm (Arkkio는 +4~7% 바이어스)
- 기어비 검산: 8.1~8.7 × 13.4 ≈ 109~117 Nm → 모델명 Peak 120 Nm 부합
- 주의: 단일 "부하 해석" 버튼=Arkkio(바이어스 있음), "부하 스윕"=가상일(정확).

### Q. "다른 모델 넣었는데 KRO80이 나온다"
프로그램 오작동 아님. 상태바가 실제 파싱 성공 파일명을 표시하는데
`InnerType_KRO80..._로드 완료`로 떠 있었음 + 변수값도 KRO80 설계와 일치.
파일 선택창에서 옆 파일(InwheelMotor 등)과 혼동했을 가능성. → 아래 2.4 제약 발견.

### Q. 폴더에 aedt가 여러 개면 첫 번째만 읽나?
구분 필요:
- **폴더에 파일 여러 개**: 문제없음. 선택창에서 고른 파일을 읽음.
- **한 파일 안에 설계(Design) 여러 개**: ⚠️ **첫 번째 설계만 읽음** (미해결).

## 3. 미해결 과제 (다음에 이어서)

### 3.1 [요청 대기] 한 파일 내 다중 Design 선택 기능
파서 `parse_aedt`가 `find_deep("Maxwell2DModel")`로 첫 설계만 잡음.
한 파일에 설계가 여러 개면 나머지 접근 불가.

| 파일 | 내부 설계 수 |
|---|---|
| SH_Reducer_QDD_14_17.aedt | **10** |
| SH_Reducer_QDD_20.aedt | 7 |
| SH_Reducer_25_32.aedt | 4 |
| 750W,1200W.aedt | 2 |
| 400W · KRO80 · InwheelMotor 등 | 1 |

→ 파일 로드 시 설계가 2개 이상이면 "어떤 설계?" 선택창 띄우기. 사용자에게
   진행 여부 확인 후 구현 예정. (QDD·750W/1200W 쓰려면 사실상 필수)

### 3.2 KRO80 DOE 전류 기준 확인 필요
현재 DOE는 **상전류 28.1A** 기준 생성. GUI 입력은 선간 46.6A(Δ)→상 26.9A.
46.6A(Δ)가 카탈로그 사양이면 DOE 전류가 4.5% 높게 잡힌 것 → 26.9A로 재생성 권장.
어느 쪽이 정격인지 사용자 확인 대기.

### 3.3 외전형(外轉形) 지원 — 보류 중
사용자 지시로 내전형 안정화 우선. 외전형 2종(SPG_X12_320,
OuterType_KRO80_Rev2)은 명확한 안내와 함께 차단 상태. 작업 전후
`scripts/run_regression.py`로 내전형 7종 회귀 보호할 것.

### 3.4 추가로 받으면 좋은 Maxwell 스크린샷 (우선순위)
1. **부하 해석 Loss Plot** (CoreLoss·SolidLoss·동손 시계열) — 철손 −25% 원인 분리,
   자석 와류손 직접 대조
2. 다른 모델(KRO80/SH_25)의 토크·EMF 플롯 + 운전조건 — 비400W 정확도
3. Maxwell 상저항 값(턴수·선경·병렬조건) — MLT 상저항 타모델 검증
4. 코깅 전용 미세스텝 해석 — 코깅 +28%가 진짜 오차인지 노이즈인지 판별
5. Mesh 통계 — B 최대값 +6.8% 편차 설명
