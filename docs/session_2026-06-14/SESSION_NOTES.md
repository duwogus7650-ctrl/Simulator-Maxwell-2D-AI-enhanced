## 1. 솔버 검증 (Maxwell/PDF 대비) — 확정값

| 모델·항목 | motoropt | 기준 | 편차 |
|---|---|---|---|
| 400W 무부하 EMF @1000rpm | 6.159 V | Maxwell 6.166 | **−0.1%** |
| 400W 부하토크(가상일) @4.9A | 0.86 N·m | PDF 0.85 | +1~2% |
| 400W 효율 @정격 | 94.7% | PDF 93.1 | +1.6%p(자석와류·기계손 제외) |
| KRO80 부하토크(가상일) @26.9A·14턴 | 7.67 N·m | Maxwell 7.13 | **+7.9%**(극한 포화 B≈3T) |
| KRO80 효율 @순시·14턴·**0.25mm** | 66.7% | 보고서 67.8 | **−1.1%p** |

- **토크는 가상일(coenergy)이 정확** — Arkkio 단일포지션은 +4~7% 과대(400W서 +8%).
  부하 스윕 T_avg를 Arkkio→가상일로 교체.
- **KRO80 토크 +7.9%**는 극한 과부하(45.6 A/mm²)·깊은 포화의 정상 변동(턴수14·전류
  ·각도id0·Br·토크법·BH외삽 다 배제 후 잔차). 적층계수는 aedt에 없어 1.0.
- **KRO80 코일선경 = 0.25mm** (사양표). 0.25로 효율 66.7%=보고서 −1.1%p 일치.
  (세션 중 0.25↔0.3 혼동 있었으나 0.25 확정. GUI 기본 0.3이라 KRO80엔 0.25 입력 필수.)
- 동손 MLT 모델: 400W −3%·KRO80 +8%(±10%). 더 정확히는 GUI 상저항에 실측 R_ph 입력.

## 2. GUI/엔진 추가 기능

- **턴수(Zc) 입력**: aedt값 자동, 무부하·부하·스윕·DOE 전체 반영(KRO80 aedt 15이나
  설계 14 → 14 입력).
- **적층계수 입력**: 강판 BH에만 적용(EMF 불변), aedt값 자동(400W 0.97).
- **코깅토크 최적화**: 무부하 −dW_co/dθ 1주기(24점) FFT 저차로 산출(raw Arkkio
  pk-pk는 슬라이딩밴드 노이즈로 3~6배 과대였음). a_m에 강하게 의존 → 학습 가능.
- **전류=동손 최소화 목표(Pcu_per_Nm2)**: 목표토크당 동손[W/Nm²], 작을수록 좋음.
  고정전류 효율은 토크와 중복이라, 이게 의미있는 손실 최적화 지표.
- **UI 한글화**: 평균토크/역기전력/자석면적/토크리플/효율/코깅토크/동손 + 방향
  (최대화↑/최소화↓/목표치◎) + 하한·타겟·상한 라벨 + 방향따라 칸 자동 활성화.
- **진행상황**: 진행바·완료 비프·버튼 잠금. **실시간 형상 모핑**(DE 세대마다 형상 갱신).

## 3. 서로게이트 학습 가능성 (DOE→최적화)

- 학습 잘 됨(목표 적합): T_avg R²0.965·emf 0.980·magnet 0.999·**cogging**(a_m의존).
- 학습 안 됨(노이즈 한계, 목표로 쓰지 말 것): ripple_pct·B_tooth (R²<0).
- efficiency: 400W 평평(0.4%p, 무의미)·KRO80 변동 크나 고정전류라 ≈토크 종속.

## 4. 자체검증 가드레일 (신뢰성 — 비전문가 안전)

이 세션의 핵심 교훈: 결과 검증을 사용자 전문성에 의존하면 위험. 도구가 스스로 잡게:

1. **데이터범위 자동충전**: 목표 L/U를 그 모델 DOE 실측 범위로 → 어떤 aedt든
   D=0 붕괴 방지(기준값 기반이면 기준이 이미 최적일 때 D=0됨).
2. **사용자 편집 보존**: 직접 설정한 목표는 자동충전이 안 덮어씀.
3. **결과 자동 진단 `diagnose_result`**: ①D=0 원인(어느 목표 0점·왜) ②AI 서로게이트
   과대예측 경고(서로D≫FEM D) ③물리 타당성(B>2.5T·코깅>토크3%·Arkkio↔가상일>8%·
   비현실 효율) → 액티브러닝 로그 + Result 탭 경고라벨.
4. **`scripts/run_validation.py`**: 검증값(400W·KRO80)을 코드에 박아 자동 대조,
   솔버 드리프트 시 FAIL. 현재 5/5 PASS.
5. **MLT 동손 추정 경고**: 정확값은 실측 R_ph 입력 안내.

전역 `~/.claude/CLAUDE.md`에 "Self-verification — assume the user won't catch errors"
섹션 추가(수치·주장은 제시 전 검증/미검증 라벨, 불확실성 선제 노출).

## 7. UI 다크테마 + 동손 W + 리플 검증 (06-15 추가)

- **UI 다크 엔지니어링 테마** (mini motor-cad 스타일): 네이비(#0b0f1a)+시안(#36cdd6)
  글로벌 QSS(`DARK_QSS`)로 전 위젯 일괄 적용. 상단 브랜드 헤더(MOTOR**OPT** +
  현재 모델명 `▣ …` 표시), 시안 밑줄 활성탭, 패널/표/입력칸 다크, 모노스페이스
  숫자, 진행바 시안. 버튼 강조: 파랑=주요(aedt열기·부하해석·액티브러닝·내보내기),
  초록=무거운 FEM(DOE·부하스윕). matplotlib도 rcParams로 캔버스 다크 통일(그리기
  코드 무수정). 캡처: `ui/dark_*.png`.
- **동손 목표 W/Nm²→W 교체 후 결국 제거**: 사용자가 W/Nm²가 헷갈린다 하여 동손[W]
  (운전점 3·I²·R_ph)로 바꿨으나, 검증해보니 **고정 운전전류에서 동손[W]는 전 설계공간
  ±2.9%(568~585W)로 사실상 상수** → 최적화 불가, 상한을 그 밑에 두면 d=0→D=0.
  물리적으로 "고정전류 동손 최소화"≡"평균토크 최대화"(같은 전류로 토크↑=토크당 동손↓).
  → **동손을 목표 목록에서 제거**(SPEC_EXTRA·Y_KEYS_OPT·자동충전). 동손 절대값은
  Solve 탭 P_cu로 확인. doe.py Pcu_per_Nm2(변동 41%, 토크와 중복)는 참고로 유지.
  교훈: 새 목표 추가 전 "그게 설계변수로 실제 변동하는가"를 먼저 검증할 것.
- **토크리플 검증**: "서로게이트 학습 불가"와 "Maxwell 대비 검증"은 별개.
  `run_validation.py`에 리플 검증 추가 — **KRO80 2.88% vs 보고서 3.0%(−0.1%p, 잘 맞음)**,
  **400W 4.65% vs 보고서 2.97%(+1.7%p, Arkkio pp 밴드노이즈로 과대)**. 리플 절대값은
  ±2%p 한계라 느슨한 허용오차(±0.5%p 드리프트)로 모니터.

## 8. 인코딩 버그 수정 (06-15) — cp949 크래시

액티브러닝이 `UnicodeDecodeError: 'cp949'`로 죽던 문제. 원인: 데이터 파일 I/O가
인코딩 미지정이라 Windows 기본 cp949로 읽음 + 실패 DOE 행의 **한글 에러 메시지**
(`"status":"fail: ValueError: 형상 불…"`). 원본은 `\u` 이스케이프(ASCII)였으나
동손 백필이 raw UTF-8로 재저장해 잠재 버그가 표면화. **근본 수정: 모든 jsonl
read/write에 `encoding="utf-8"`** (surrogate.load_dataset, doe.build_dataset
read/append, app.py 액티브러닝 append, run_p5_doe·run_p6_active). 로케일·내용
무관하게 안전. 검증: load_dataset 정상(X(62,5)).

## 9. 라운드 카운터 + 하드 제약(필수) + AI 권장 목표값 (06-15)

- **액티브러닝 라운드 카운터**: `self._active_round`(모델별, 리셋). 버튼·로그·
  진행바에 "N라운드" 표시(이전엔 매번 "1라운드"). `_cand_done`에 "누적 후보 N개·
  현재 최고 D=… (라운드 나빠도 최고 유지)" 표시 — 후보는 D순 정렬, 1번이 최고.
- **하드 제약(필수 🔒) 열**: Objective 표에 7번째 "필수" 체크박스. 체크 시 그 목표는
  통과/실패(만족하면 1, 어기면 D=0 탈락), 소프트는 기존 만족도 램프.
  `objective.py`: `_hard_pass`, `desirability(...,hard_keys)`,
  `desirability_from_dict(...,hard_keys)`, `SurrogateObjective(hard_keys)`.
  GUI: `_hard_keys_from_table()`, run_active_round·_update_result·diagnose_result에
  전달. 예: 토크 7130 필수→ 7130↑면 d=1(토크 램프 패널티 없음)→ 옵티마이저가
  코깅·리플 최적화에 집중(7300서 D 0.46→0.80). DE 탐색·FEM 검증 모두 하드 반영.
- **AI 권장 목표값 버튼**: 목표가 빡빡해 D=0일 때, **필수(🔒)는 고정**하고 소프트
  목표 L/U를 "필수 만족 설계의 DOE 달성범위"로 조정 제안 → QMessageBox로 변경값
  미리보기 + Yes/No(사용자 선택). 필수 자체가 데이터서 불가능하면 각 필수의 달성
  범위를 안내(완화/DOE확대 권장). `relax_to_recommended()`.
- diagnose_result: D=0 원인을 하드 위반/소프트 0점 구분, 'AI 권장 목표값' 버튼 안내.

**최적화 엔진 정리(사용자 질문)**: 서로게이트(MLP 신경망) + **차분진화 DE(진화
알고리즘)** + 액티브러닝(DE해→FEM검증→재학습 반복). RL 아님(SAC는 미학습 옵션).
후보 표의 EMF·자석 열은 목표 아닌 **참고 표시값**(체크한 목표만 D에 반영).
